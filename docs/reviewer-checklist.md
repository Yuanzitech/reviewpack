# Reviewer Checklist

Reviewpack can generate a deterministic reviewer checklist for maintainers.

This checklist helps reviewers turn Reviewpack analysis into concrete review steps.

It does not call AI providers.

It does not replace human judgment.

## Output

Reviewpack writes:

    .reviewpack/reviewer-checklist.md

This file is generated by default together with the main review pack outputs.

## Commands

From fixture input:

    reviewpack from-fixture examples/fixtures/simple-pr.json --output .reviewpack

From local git diff:

    reviewpack local --base main --head HEAD --output .reviewpack

From GitHub pull request metadata:

    reviewpack github https://github.com/owner/repo/pull/123 --output .reviewpack

## What it contains

The reviewer checklist may include sections such as:

- Core Review
- Compatibility
- Tests
- Documentation
- Dependencies
- CI
- Infrastructure
- Risk
- Release
- Privacy

Each checklist item includes:

- A checkbox
- A concrete review task
- A reason explaining why the item was generated

## Example

    # Reviewer Checklist

    ## Checklist

    ### Core Review

    - [ ] Review behavior changes and edge cases.
      - Reason: Source files changed.

    ### Tests

    - [ ] Confirm whether test coverage is sufficient.
      - Reason: Source files changed without test file updates.

    ### Release

    - [ ] Decide whether this PR needs a changelog or release note entry.
      - Reason: Maintainers should explicitly decide release note impact.

## What it does not do

The reviewer checklist does not:

- Call AI providers
- Upload source code
- Upload raw diffs
- Approve pull requests
- Merge pull requests
- Post PR comments
- Replace maintainers

## Privacy behavior

The reviewer checklist is generated from Reviewpack analysis results.

It does not require:

- AI provider tokens
- Raw diffs
- Full source code
- Branch names
- Commit messages

GitHub mode may use network access to fetch PR metadata and changed file statistics, but the reviewer checklist itself is generated locally from the resulting analysis.
