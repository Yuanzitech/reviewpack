# Release Note Hints

Reviewpack can generate deterministic release note hints for maintainers.

This feature helps maintainers decide whether a pull request should be mentioned in release notes or changelogs.

It does not call AI providers.

It does not generate final release notes automatically.

## Output

Reviewpack writes:

    .reviewpack/release-note-hints.md

This file is generated by default together with the main review pack outputs.

## Commands

From fixture input:

    reviewpack from-fixture examples/fixtures/simple-pr.json --output .reviewpack

From local git diff:

    reviewpack local --base main --head HEAD --output .reviewpack

From GitHub pull request metadata:

    reviewpack github https://github.com/owner/repo/pull/123 --output .reviewpack

## What it contains

The release note hints file may include:

- Possible release categories
- Maintainer checklist
- Dependency change hints
- CI change hints
- Documentation change hints
- High-risk area hints
- Privacy notes

## Example categories

Reviewpack may suggest categories such as:

- Changed
- Dependencies
- CI
- Infrastructure
- Documentation
- Tests
- Risk
- Review

These categories are deterministic hints based on changed files and risk signals.

## What it does not do

Release note hints do not:

- Call AI providers
- Upload source code
- Upload raw diffs
- Write final release notes automatically
- Decide whether a PR must be included in a release
- Replace maintainer judgment

## Maintainer workflow

A typical workflow:

1. Generate a Reviewpack output.
2. Open `.reviewpack/release-note-hints.md`.
3. Review possible release categories.
4. Decide whether the PR needs a changelog entry.
5. Write final release notes manually.

## Privacy behavior

Release note hints are generated from local Reviewpack analysis results.

They do not require:

- AI provider tokens
- Raw diffs
- Full source code
- Branch names
- Commit messages

GitHub mode may use network access to fetch PR metadata and changed file statistics, but release note hints themselves are generated locally from the resulting analysis.
